# Prometheus

> Open-source systems monitoring and alerting toolkit originally built at SoundCloud and now a graduated CNCF project. Prometheus scrapes and stores metrics as time series, and provides the PromQL query language for analysis, alerting, and dashboarding. This is an API Evangelist catalog profile; llms.txt is generated from the repo (the project does not publish its own).

## APIs

- [Prometheus HTTP API](https://prometheus.io/docs/prometheus/latest/querying/api/): Instant and range PromQL queries, label/series/metadata lookup, targets, rules, alerts, and TSDB status under /api/v1.
- [Prometheus Management API](https://prometheus.io/docs/prometheus/latest/management_api/): Administrative lifecycle endpoints — config reload, TSDB snapshot, delete series, clean tombstones, graceful shutdown (require --web.enable-lifecycle / --web.enable-admin-api).
- [Prometheus Pushgateway API](https://github.com/prometheus/pushgateway): Accepts metrics pushed from short-lived batch jobs, grouped by job and labels.
- [Prometheus Alertmanager API](https://prometheus.io/docs/alerting/latest/alertmanager/): Alertmanager v2 API for alert status, silences, receivers, and cluster status.
- [Prometheus Remote Write API](https://prometheus.io/docs/specs/prw/remote_write_spec/): Protocol for shipping time series to remote storage via Snappy-compressed protobuf over HTTP POST.
- [Prometheus Client Libraries](https://prometheus.io/docs/instrumenting/clientlibs/): Official instrumentation libraries for Go, Java/Scala, Python, Ruby, and Rust.
